# docker 개념
## docker 란?
- 컨테이너 기반의 오픈소스 가상화 플랫폼
- 즉, 애플리케이션을 독립된 환경에서 실행할 수 있게 해주는 도구
- 내 개발 환경을 통으로 말아서 전달할 수 있게 해주는 도구
- 아파트 안에서 원룸을 만들어주는 관리인.
- 커널은 공유하되, 파일 시스템과 라이브러리는 분리


### docker, 가상머신 차이
가상머신은 독립된 운영체제를 가지고 있어 무겁고 오래걸림. 별도 커널이 있음.
```bash
                물리 서버
─────────────────────────────────
           Hypervisor
─────────────────────────────────
        VM1            VM2
┌──────────────┐  ┌──────────────┐
│ App          │  │ App          │
├──────────────┤  ├──────────────┤
│ Ubuntu       │  │ CentOS       │
├──────────────┤  ├──────────────┤
│ Linux Kernel │  │ Linux Kernel │
└──────────────┘  └──────────────┘
```
반면, 도커는 "커널이 하나!"
host 운영체제의 커널을 모든 컨테이너가 공유함.
그렇기때문에 host에서 프로세스가 돌아감.
도커는, 리눅스의 프로세스를 격리해서 실행함.

- debian : ls, cat, bash, apt 같은 프로그램 집합체
- kernel : cpu, memory, process 관리, 네트워크, 디스크 담당
```bash

          EC2(host) Ubuntu
──────────────────────────────────
          Linux Kernel
──────────────────────────────────
      Docker Engine (dockerd)
──────────────────────────────────
   nginx            alpine
┌─────────────┐  ┌─────────────┐
│ nginx       │  │ sh          │
├─────────────┤  ├─────────────┤
│ Debian      │  │ Alpine      │
│ User Space  │  │ User Space  │
└─────────────┘  └─────────────┘
```

# docker compose
## 1. docker compose란?
compose : 컨테이너 실행 정보를 yaml 파일로 저장하는 도구
명령어 대신 yaml을 작성하여 실행.

### 필요성
컨테이너를 실행할 때 컨테이너별로 옵션이 많고 각각 실행시켜야하는 번거로움이 있음.
db, redis, spring.. 등등 매번 계속 아래처럼 docker run을 해줘야함.
하지만 compose를 사용하면 필요한 것들을 yaml 파일로 작성한뒤 compose하면 한번에 가능.
```bash
# docker run
 docker run -d \
--name web \
-p 80:80 \
-v ~/html:/usr/share/nginx/html \
--network my-network \
nginx

docker run ..... 을 컨테이너별로 매번 실행해야함.
```

```bash
# docker compose yaml 작성
services:
  web:
    image: nginx
    ports:
      - "80:80"

    volumes:
      - ./html:/usr/share/nginx/html

    networks:
      - my-network

# yaml 파일 실행
docker compose up -d
```

## 2. 단일 comtainer compose 실습

### compose 설치 확인

```bash
docker compose version

# 미설치시 compose 설치
sudo apt update

sudo apt install docker-compose-v2
```

### 프로젝트 폴더 안에 yaml 파일 생성
compose는 현재 폴더를 기준으로 하기 때문에 주로 프로젝트 폴더에서 yaml 파일을 작성한다.
-> compose에서는 폴더 하나가 하나의 프로젝트 개념.
```bash
spring-mysql/
├── docker-compose.yml
├── app/
│   └── Dockerfile
└── mysql/
    └── init.sql

이 폴더 안에서 docker compose하면 app, db가 같이 올라감.
```

```bash
# yml 파일 생성
services:
  web:
    image: nginx

    container_name: web

    ports:
      - "80:80"

    volumes:
      - ./html:/usr/share/nginx/html -> docker-compose.yml 파일이 있는 위치 기준으로 작성
```

### compose 실행
compose하면 기본적으로 현재 폴더에서 "docker-compose.yml", "compose.yml"을 자동으로 찾기 때문에 yml 파일명을 작성하지 않아도 알아서 찾아 실행함.
```bash
# yml 파일 이름이 docker-compose, compose 일때
docker compose up -d

# yml 파일 이름이 다른거로 되어있을 때
# -f 파일 옵션을 직접 지정해야함
docker compose -f 파일명.yml up -d
```

### 실행 후 확인
docker compose 하면 docker 내부적으로 컨테이너, 네트워크, 볼륨을 생성해줌
```bash
Compose Project
        │
        ├── web 컨테이너
        ├── compose-study_default 네트워크
        └── (필요하면 Volume)
```

### compose down
compose가 만든 환경을 정리
자기가 만든 리소스를 정리하는 명령어. stop 과는 다름!

| 명령어                    | 컨테이너    | 네트워크 | Volume |
| ---------------------- | ------- | ---- | ------ |
| `docker compose stop`  | ⏸ 정지만   | 유지   | 유지     |
| `docker compose start` | ▶ 다시 시작 | 유지   | 유지     |
| `docker compose down`  | ❌ 삭제    | ❌ 삭제 | 유지(기본) |

```bash
# 명령어
docker compose down
-> 이 명령어로 지우면 volume은 삭제되지 않음

Compose Project
        │
        ├── web 컨테이너      ❌ 삭제
        ├── compose-study_default 네트워크  ❌ 삭제
        └── Volume            (기본적으로 유지)

# 명령어
docker compose down -v 
-> 볼륨까지 지우려면 이 명령어 써야하는데 조심해야됨..단, named로 docker가 생성한 볼륨일 경우만 삭제
-> bind의 경우 삭제되지 않음.

Compose Project
        │
        ├── web 컨테이너      ❌ 삭제
        ├── compose-study_default 네트워크  ❌ 삭제
        └── Volume   ❌ 삭제
```

| 명령                       | Container | Network | Bind Mount | Named Volume |
| ------------------------ | --------- | ------- | ---------- | ------------ |
| `docker compose down`    | ❌ 삭제      | ❌ 삭제    | ✅ 유지       | ✅ 유지         |
| `docker compose down -v` | ❌ 삭제      | ❌ 삭제    | ✅ 유지       | ❌ 삭제         |



## 3. 다중 comtainer compose 실습

1. yaml 파일 작성
```bash
# yml 파일 생성
services:
  web:
    image: nginx
    container_name: web
    ports:
      - "80:80"
    volumes:
      - ./html:/usr/share/nginx/html

  client:
    image: alpine
    container_name: client
    tty: true -> 컨테이너를 종료하지 말고 계속 켜놓으라는 의미
```

2. compose 실행 및 확인
```bash
# 컨테이너 실행 확인
docekr ps

# 결과

f9e82267c494   alpine    "/bin/sh"                2 minutes ago   Up 2 minutes    

# 네트워크 확인
docker network ls

# 결과
NETWORK ID     NAME                    DRIVER    SCOPE
6cc2fbdd5eba   bridge                  bridge    local
fd4ad329ca8e   compose-study_default   bridge    local -> compose로 생긴 네트워크

87b5122a3180   my-network              bridge    local
8fd0839bf891   none                    null      local
```

3. dns 되는지 확인하기
- Docker Compose는 프로젝트별 전용 네트워크를 자동으로 생성
- Docker의 내장 DNS를 이용해 같은 네트워크에 있는 컨테이너들이 이름으로 서로 통신할 수 있도록 설정
  -> 따라서 ip 없이 dns name으로 호출이 가능!
```bash
# 컨테이너 내부로 접속
docker compose exec client sh

# 다른 컨테이너로 ip 없이 호출해보기
ping web
wget -O- http://web
```

```bash
컨테이너 생성
    ↓
같은 Network 연결
    ↓
Docker DNS에 이름 등록
```

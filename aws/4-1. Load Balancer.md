# 로드밸런서
## 로드밸런서란?
인터넷에서 들어오는 요청을 어디로 보낼지 결정하는 장비
NAT와는 방향이 반대.

## 필요성
트래픽이 많아졌을 때 하나의 서버로 감당하기 어려운 상황이 됨.
이때 ec2를 늘리게 되면 각각의 ec2로 트래픽을 보내기 위한 설정이 필요함.
dns에 ec2 ip를 등록해도 도메인으로 접속은 가능하지만 아래와 같은 문제가 있음.

1. 여러 서버 중 1대가 죽었을 경우
   dns는 서버가 죽었다는 것을 알 수 없기 때문에 특정 사용자들은 죽은 서버로 접속하게 되는 문제 발생
2. 특정 서버에 리소스 사용량이 높은 경우
   dns는 특정 서버의 cpu, memory 사용률이 높은 것을 알 수 없기 때문에 이를 고려해서 트래픽을 분산할 수 없음.
   따라서 특정 사용자들은 리소스가 과도하게 사용 중인 서버로 접속하게 되는 문제 발생
3. url에 따라 다른 서버로 트래픽을 보내고 싶은 경우
   /shop, /api 등 url에 따라 다른 서버로 트래픽을 보내고 싶을 때 dns는 해당 기능이 없음.

* AWS ALB는 사용자가 도메인으로 접속하기만 하면, 서버가 죽었는지 체크하고 사용량이 많으니 체크해서 알아서 트래픽을 분산해줌.
* url별로 다른 서버로 트래픽을 보낼 수도 있음.

## 특징

```bash
   Internet
      │
      ▼
Load Balancer
   ┌──┴──┐
   ▼     ▼
EC2-1  EC2-2
```
1. 트래픽 분산
   한 서버에 과도하게 트래픽이 몰리지 않게 분산시킴
2. 장애 감지
   죽은 서버는 제외하고 트래픽을 분산함
3. 고가용성
   서버 한 대가 죽어도 서비스는 멈추지 않음
4. 확장성
   서버가 많이 늘어나도 사용자는 도메인으로 접속하면 됨

## aws 로드밸런서
aws에서는 새가지 로드밸런서를 지원하고 있음.
ALB가 가장 많이 쓰이고, NLBsms TCP, GWLB는 특수한 경우에 사용함.
| 종류   | 계층             | 언제 사용?      |
| ---- | -------------- | ----------- |
| ALB  | L7(HTTP/HTTPS) | 웹 서비스       |
| NLB  | L4(TCP/UDP)    | 게임, DB, 초고속 |
| GWLB | L3/L4          | 방화벽, 보안 장비  |

# NAT

## NAT 란?
- network address translatiohn으로 주소를 변환하는 장비 
- NAT는 public ip(EIP)를 가지고 있음
- 들어오는 연결을 전달하는 장비가 아니라, "나가는 연결의 응답"만 돌려주는 장비
  "내부에서 먼저 시작한 연결"에 대한 응답만 돌려주는 장비
   -> internet gatway와 차이점
   -> 보안이 좋음

## 필요성
- private ec2에서 인터넷을 사용하기 위해
```bash
Public EC2
    │
    ├── 인터넷 가능 ✅
    └── SSH 가능 ✅

Private EC2
    │
    ├── 인터넷 불가 ❌
    └── SSH (Public EC2를 통해서만) 가능 ✅
```
NAT로 주소를 변환하면 인터넷에서는 public ip만 보임.
요청에 응답할 때도 NAT를 통해 다시 private ip로 전환해줌
```bash
10.0.2.15
    ↓
52.xx.xx.xx  -> NAT가 private ip(10.0.1.xx)를 public ip(52.xx.xx.xx)로 변환
    ↓
 8.8.8.8   
```

## 실습

### 1. EIP 생성
NAT gatewaysms 고정된 Public ip가 필요하기 때문에 EIP를 생성해야함.

### 2. NAT gateway 생성
nat도 결국 외부 인터넷이랑 연결되어야하기 때문에 반드시 public subnet에 만들어야함.

### 3. private subnet에 연결된 route table에 NAT 추가
public route table에서는 internet gateway를 추가했지만, private에는 NAT를 추가해야함.

### 4. private 서버에서 외부로 접근해보기
private ec2는 public ip가 생기지 않음. 언제까지나 NAT gateway를 통해서 인터넷과 통신!
```bash
ping 8.8.8.8
-> 성공

curl https://checkip.amazonaws.com
-> 성공
-> 해당 명령어는 aws 서버에 접속해서 현재 인터넷에 보이는 public ip 확인
-> 이때 보여지는 Ip는 NAT에 연결된 EIP.
-> public ec2에서 실행하면 ec2의 public ip가 나옴
```

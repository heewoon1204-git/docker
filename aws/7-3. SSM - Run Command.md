# SSM - Run Connmand 중심

## Run Connmand란?
SSH 접속 없이 ec2에 명령을 실행하는 기능
```bash
AWS Console
    ↓
명령 입력
    ↓
EC2 실행
```

## 필요성
예를 들어 서버 100대에 동일한 작업을 수행해야할 때 서버 하나하나 실행하면 귀찮고 오래걸림.
그때 이 기능으로 한번에!
```bash
 100대
  ↓
로그 수집
  ↓
서비스 재시작
  ↓
패키지 업데이트
  ↓
설정 변경
```

## 실습!!

1. Run Command
SSM -> 노드 도구 -> 명령 실행

2. Document 실행
```bash
Document : AWS-RunShellScript
  -> 리눅스에서 shell 명령 실행하는 document
명령 파라미터 : 실행하고 싶은 명령어 쓰기
            예시)
            pwd
            echo "===== HOSTNAME ====="
            hostname

            echo "===== IP ====="
            hostname -I

            echo "===== UPTIME ====="
            uptime
Working Directory : /
   -> 명령을 어느 폴더에서 실행할 것인지 작성 (cd working directory 명령어 한줄이 추가된다고 보면 됨)
   -> 작성 안하면 default값이 / 혹은 /home/ssm-user 
Execution Timeout : 3600(기본값)
   -> 명령이 몇 초 동안 실행될 수 있는가를 의미
   -> 해당 시간이 지나면 ssm이 종료시킴
```
